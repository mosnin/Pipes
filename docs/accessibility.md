# Accessibility Notes

## Current posture
- Keyboard focus styles are consistently visible on buttons, links, and form controls.
- Inputs for dashboard, onboarding, and settings actions include explicit labels/aria labels.
- Empty states now use polite status regions for assistive technologies.
- Settings navigation uses semantic `nav` structure.

## Remaining limits
- Editor canvas interactions remain mouse-first for advanced node manipulation.
- Full WCAG contrast audit and screen-reader journey audit are still pending for launch+1.
